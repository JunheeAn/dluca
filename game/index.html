Project LUA

Real Presence, Real Conversation From answers to encounters

On-Screen Dialogue NPC Toolkit (보이는 대화 NPC 툴킷). UE5/MetaHuman 기반으로 대화·자막·음성·립싱크·감정을 한 프레임 안에서 맞물리게 만들어 ‘답변’이 아니라 ‘만남’을 구현합니다.

Problem → Opportunity

Presence Gap (존재 격차) — 텍스트/음성만으로는 신뢰·정서·몰입 축적이 어렵습니다.

Uncanny Fatigue (불쾌부조화 피로) — 표정·타이밍이 어긋나면 피로도가 급증합니다.

Brand Incoherence (브랜드 일관성 붕괴) — 장면별 톤/금칙/지식 경계 유지가 어렵습니다.

Operational Friction (운영 마찰) — 응답 속도·백채널·표현 미세조정에 높은 비용이 듭니다.

Game fit — 튜토리얼/가이드/도슨트/라이브 이벤트 장면에서 완료율·체류시간·설득력 개선이 목표입니다.

Tutorial Completion ↑ Quest Acceptance ↑ Session Length ↑ Player Trust ↑

Solution — On-Screen Dialogue NPC Toolkit

LUA는 팀의 아트/연출을 Dialogue Model(대화 모델), TTS(음성 합성), Lip-sync/Emotion(립싱크/감정)과 룰 기반으로 묶어주는 런타임 툴킷입니다. 스튜디오는 장면/튜토리얼에 맞춰 NPC Persona Preset(페르소나 프리셋)과 Quest/Scene Script(퀘스트/씬 스크립트)만 교체하면 됩니다.

  • Visible Conversation(보이는 대화): 시선·표정·백채널·타이밍까지 화면에서 보이게.
  • Persona Preset(페르소나 프리셋): 톤/금칙/지식 경계 프로필을 데이터 에셋으로 관리.
  • Scene Script(씬 스크립트): 튜토리얼/도슨트용 대화 규칙셋, 페이싱 버짓 포함.
  • Ops-Ready(운영 대비): 폴백/로그/리플레이/프리셋 스위치로 라이브 대응.

Runtime Director (런타임 디렉터)

Runtime Interaction Flow

Input → Dialogue Model(stream) → TTS → Viseme 계산 → Lip-sync + Emotion/Micro-Idle → Subtitle/Audio Sync

Dialogue Director Rules (대화 디렉터 규칙)

발화 끊김을 줄이기 위해 합성 시작은 두 조건으로 제한합니다: (a) 최종 문장 확정(OnFinal) 수신, (b) 짧은 시간/길이 기준으로 청크가 임계치 이상 누적(짧은 청크 결합).

  • Viseme smoothing: attack/release, low-pass로 입모양 튐 억제
  • Emotion easing: 표정 전이 이징으로 프레임 감성 유지
  • Gaze IK(시선 동기화): 카메라 랜드마크(u,v) → Eye/Head 본 매핑(+스무딩)
  • UI State: Listening/Thinking/Speaking, 백채널 패턴 라이브러리
  • Safety Guard: 금칙/필터, 폴백/로그/리플레이

Game Terms (한/영 병기) & Presets

  • On-Screen Dialogue (보이는 대화)
  • Runtime Director (런타임 디렉터) — 대화/자막/발화의 시점·전이 규칙
  • NPC Persona Preset (NPC 페르소나 프리셋) — 톤/금칙/지식 경계
  • Quest/Scene Script (퀘스트/씬 스크립트) — 장면별 규칙셋
  • Timing Budget (타이밍 버짓) — 자막/발화/백채널 타이밍 예산
  • Gaze IK (시선 동기화)
  • Safety Guard (콘텐츠 필터/폴백)
  • Presence Gap / Uncanny Fatigue

Preset 구조(예시)

  • persona.json — tone, forbidden, knowledge boundary
  • scene.json — tutorial/guide rules, timing budget
  • emotion.json — curve constants, transition rules
