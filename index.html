Project LUA

Overview

Project LUA는 ‘보이는 대화(Visible Conversation)’를 목표로 하는 실시간 디지털 휴먼 연구/프로토타입입니다. 로컬 환경의 언어 모델이 대화 의도와 맥락을 생성하면, 시스템은 이를 자막·음성·입모양·감정·미세동작으로 정교하게 동기화해 마치 사람과 마주 앉아 대화하는 듯한 존재감 있는 상호작용을 만듭니다. 핵심 가치는 ‘더 빠른 답’이 아니라 관계감과 설득이며, 교육·전시·상담 등 현장형 인터랙션에서의 효용을 검증하고자 합니다.

Problem

  • Presence Gap — 텍스트/음성 챗봇은 응답은 하지만, 시선·표정·턴테이킹이 없어 대화의 체감이 약합니다.
  • Sync Gap — 자막·TTS·입모양이 어긋나면 피로와 이질감이 누적됩니다(일명 uncanny fatigue).
  • Governance Gap — 브랜드 톤·정책 준수·재현 가능성(버전/시드) 관리가 어렵습니다.
  • Operational Constraints — 개인정보/지연/비용 제약으로 클라우드 전제의 상용 운영이 쉽지 않습니다.

Solution

  • 통합 실시간 파이프라인 — LLM 스트리밍 → 인크리멘털 TTS → 비지므(입모양) 기반 립싱크 → 감정 레이어/마이크로 아이들 → 자막·오디오 동기화. (Next: STT 입력, Gaze Sync)
  • Control & Orchestrability — 시나리오 락, 감정 강도/길이/주제/타이밍 파라미터, Operator-in-the-loop, 버전·시드 고정, 로그/리플레이.
  • Persona & Scenario Templates — 말투·금칙·지식 경계가 정의된 페르소나와 상태 전환형 시나리오를 코드 수정 없이 교체.
  • Local-first — 로컬 운용을 기본으로 하여 보안·지연·비용 리스크를 낮추고, 전시/키오스크 등 현장 적용을 고려.

System

  1. Input Layer — UI 텍스트(→ 예정: STT)
  2. Adapter — LLM 스트림 수신/분절, 상태 머신(Idle↔Listen↔Speak)
  3. LLM — 토큰 스트리밍 / 정책·페르소나 적용
  4. TTS — 인크리멘털 합성 / 타임코드 추출
  5. Viseme — 음소→비지므 맵핑, 립싱크 커브 생성
  6. Emotion — 대화 태그→표정/헤드/바디 파라미터
  7. Micro-Idle — 미세 시선·호흡·깜빡임 스케줄러
  8. Subtitle/Audio Sync — 화면/오디오 타임라인 정합
  9. Logging & Replay — 세션 기록, 재현/검수

Differentiators

  • Control & Orchestrability — 정책 기반 세밀 제어, 시나리오 락, 운영자 개입, 재현성 확보.
  • Persona & Scenario Templates — 톤·금칙·지식 경계와 상태 전환을 프로필로 분리, 재사용/교체 용이.
  • Visible Conversation UX — 립싱크·감정·미세동작 동기화로 존재감과 설득력 향상.

Roadmap (90 days)

  • M1 — STT 입력, Gaze Sync(2D 랜드마크→시선/헤드 에임), 90–180s 루프.
  • M2 — PoC(1–2곳) 적용, 패키징 α(키오스크 프로필), 로그/리플레이, 사용자 소규모 평가.
  • M3 — 패키징 β, 운영 미니패널, HCI Creative Award · Red Dot 제출.
