Real Presence, Real Conversation From answers to encounters

Project LUA는 ‘보이는 대화(Visible Conversation)’를 목표로 하는 실시간 디지털 휴먼 연구/프로토타입이다. 로컬 환경의 언어 모델이 대화 의도와 맥락을 생성하면, 시스템은 이를 자막·음성·입모양·감정·미세동작으로 정밀하게 동기화해 사람과 마주 앉아 대화하는 듯한 존재감·설득력을 만든다. 목표는 ‘빠른 답변’이 아니라, 실제 상호작용에서 체감되는 관계감을 끌어올리는 것이다.

Definition

Project LUA는 ‘존재감 자체가 인터페이스’라는 자신의 명제를 실시간 상호작용으로 구현한 디지털 휴먼 실험이자 작품이다. 텍스트 상자 속 대답을 넘어, 시선·표정·타이밍·말투와 같은 가시적 대화(Visible Conversation)를 통해 사람과 만나는 경험을 설계한다. 브랜드-정합적 톤과 정책을 유지하면서도 리얼리즘 아트(3D·조명·카메라·UI)와 엔지니어링(LLM·TTS·립싱크·자막 스트림)이 한 몸처럼 작동하도록 만든다.

Problem

오늘의 많은 AI는 답을 내놓지만 ‘만나지’는 못한다. 텍스트와 음성만으로는 신뢰·정서·몰입이 축적되기 어렵다(존재 격차, Presence Gap). 어긋난 표정과 타이밍은 피로와 거부감을 키운다(불쾌부조화 피로, Uncanny Fatigue). 즉석으로 구동되는 LLM은 브랜드의 톤·금칙·지식 경계를 일관되게 유지하기 힘들다(브랜드 일관성 붕괴, Brand Incoherence), 그리고 현장 맥락에 맞춘 응답 속도·백채널·표현의 미세 조정은 여전히 높은 운영 비용을 요구한다(운용 마찰, Operational Friction). 그 결과 AI와의 상호작용은 ‘보이지 않는 대화’에 머물고, 설득과 관계 형성으로 이어지지 못한다. — 지금 필요한 것은 보이는 대화와 대면 가능한 존재감이다.

Solution

LUA는 브랜드-맞춤형 리얼리즘 디지털 휴먼으로, 텍스트 중심 상호작용의 한계를 넘어 보이는 대화를 제공한다. 일관된 E2E 런타임 체계(LLM → TTS → Lip-sync/Emotion → Subtitles/Chat UI) 위에 페르소나·시나리오 정책을 분리해 톤·금칙·지식 경계를 유지하고, 리얼리즘 아트(모델·조명·카메라·UI)와 결합해 말과 표정·타이밍이 함께 움직이도록 설계한다. 이 구성은 Presence Gap을 줄이고, Uncanny Fatigue를 낮추며, Brand Incoherence를 제어한다. 또한 모듈형 어댑터와 프로필 기반 운영으로 Operational Friction을 완화해, 키오스크·도슨트·튜터 등 다양한 장면으로 확장 가능한 기반을 마련한다.

System Architecture

Runtime Interaction Flow

Input → LLM(stream) → TTS → Viseme 계산 → Lip-sync + Emotion/Micro-Idle → Subtitle/Audio Sync

Orchestration

LUA의 오케스트레이션은 수치 튜닝 이전에 흐름을 표준화하는 데 초점을 둔다. 스트리밍 수신 시 자막은 실시간으로 업데이트하고, 음성 합성 시작은 두 조건으로 제한한다: (a) 최종 문장 확정(OnFinal) 수신 시점, 또는 (b) 짧은 시간/길이 기준으로 청크가 일정량 모였을 때(‘짧은 청크 결합’ 버퍼 임계치 충족). 이렇게 해서 잘게 끊기는 발화를 방지한다.

합성된 오디오는 OVRLipSync → AnimBP(ARKit 커브)로 전달되어 입모양–음성 동기를 맞추고, viseme 스무딩(attack/release, low-pass)과 Emotion 이징으로 표정/입모양의 미세 튐을 억제한다. Blink 정상화·Micro-Idle·UI Reveal은 대기→청취→발화 전환 구간을 메워 상호작용의 연속성을 확보한다.

시작 상태는 WBP_Subtitle → WBP_Chat 순 레이어링과 커서/클릭/오버 활성화로 즉시 조작 가능하게 열어 두며, 텍스트 파이프라인에서는 어댑터 단계 정제로 불필요한 문자열이 음성으로 읽히지 않도록 한다. (STT·Gaze Sync는 동일한 프레임에 편입될 로드맵 항목으로 유지.)

11월 — 시스템 품질 정비

  • STT 입력 도입(청취 상태·침묵 타임아웃·발화 전환 규칙)
  • Gaze Sync 1차(카메라 랜드마크 u,v → 아이/헤드 본 매핑 + 기본 스무딩)
  • UI 상태 체계화: Listening / Thinking / Speaking, 백채널 패턴 라이브러리화
  • 전체 성능 개선
  • LLM 교체 가능 구조(어댑터/프롬프트 프로필 분리)
  • 반응 속도 개선(청크 결합, 자막/발화 페이싱 버짓 도입)
  • Lip-sync 튐 완화(viseme attack/release·LPF·이징)
  • 표정 개선(Emotion 커브 시간상수·전이 규칙 정립)

12월 — 패키징 & 평가

  • 패키징/런처·프리셋 번들(데모/전시용 프리셋, 로깅 옵션)
  • 사용자 소규모 평가(파일럿): 존재감·불쾌부조화·이해도 지표 설계 및 1차 수집
  • 공모전 제출: Creative Award 등 출품 자료 정리

1월 — PoC & 현장 운용 설계

  • PoC 기획/세팅 가이드(하드웨어, 카메라/마이크, 네트워크, 공간 조도)
  • 운영 시나리오(오프라인 폴백, 정책/시나리오 롤백, 로그/리플레이)
