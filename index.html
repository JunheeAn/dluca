Project LUA

Overview

Project LUA는 ‘보이는 대화(Visible Conversation)’를 목표로 하는 실시간 디지털 휴먼 연구/프로토타입입니다. 로컬 환경의 언어 모델이 대화 의도와 맥락을 생성하면, 시스템은 이를 자막·음성·입모양·감정·미세동작으로 정밀하게 동기화해 사람과 마주 앉아 대화하는 듯한 존재감·설득력을 만듭니다. 목표는 ‘빠른 답변’이 아니라, 실제 상호작용에서 체감되는 관계감을 끌어올리는 것입니다.

Demo: YouTube · Vision: YouTube · EdTech: YouTube · Executive Summary (PDF): open

Problem

  • Presence Gap — 텍스트/음성 챗봇은 응답은 하지만 시선·표정·턴테이킹이 없어 대화의 체감이 약합니다.
  • Sync Gap — 자막·TTS·입모양이 어긋나면 피로와 이질감이 누적됩니다(일명 uncanny fatigue).
  • Governance Gap — 브랜드 톤·정책 준수·재현 가능성(버전/시드) 관리가 어렵습니다.
  • Operational Constraints — 보안/지연/비용 제약으로 클라우드 전제의 상용 운영이 쉽지 않습니다.

Solution

  • 통합 실시간 파이프라인 — LLM 스트리밍 → 인크리멘털 TTS → 비지므(입모양) 기반 립싱크 → 감정 레이어/마이크로 아이들 → 자막·오디오 동기화. (Next: STT 입력, Gaze Sync)
  • Control & Orchestrability — 시나리오 락, 감정 강도/길이/주제/타이밍 파라미터, Operator-in-the-loop, 버전/시드 고정, 로그/리플레이.
  • Persona & Scenario Templates — 말투·금칙·지식 경계가 정의된 페르소나와 상태 전환형 시나리오를 코드 수정 없이 교체.
  • Local-first — 로컬 운용을 기본으로 하여 보안·지연·비용 리스크를 낮추고, 전시/키오스크 등 현장 적용을 고려.

System Architecture

  1. Input Layer — UI 텍스트(→ 예정: STT)
  2. Adapter — LLM 스트림 수신/분절, 상태 머신(Idle↔Listen↔Speak)
  3. LLM — 토큰 스트리밍 / 정책·페르소나 적용
  4. TTS — 인크리멘털 합성 / 타임코드 추출
  5. Viseme — 음소→비지므 맵핑, 립싱크 커브 생성
  6. Emotion — 대화 태그→표정/헤드/바디 파라미터
  7. Micro-Idle — 미세 시선·호흡·깜빡임 스케줄러
  8. Subtitle/Audio Sync — 화면/오디오 타임라인 정합
  9. Logging & Replay — 세션 기록, 재현/검수

Runtime Interaction Flow

Input → LLM(stream) → TTS → Viseme 계산 → Lip-sync + Emotion/Micro-Idle → Subtitle/Audio Sync

Orchestration

LUA의 오케스트레이션은 수치 튜닝 이전에 흐름을 표준화하는 데 초점을 둡니다. 스트리밍 수신 시 자막은 실시간으로 업데이트하고, 음성 합성 시작은 두 조건으로 제한합니다: (a) 최종 문장 확정(OnFinal) 수신 시점, (b) 짧은 시간/길이 기준으로 청크가 일정량 모였을 때(‘짧은 청크 결합’ 버퍼 임계치 충족). 이렇게 해서 잘게 끊기는 발화를 방지합니다.

합성된 오디오는 OVRLipSync → AnimBP(ARKit 커브)로 전달되어 입모양–음성 동기를 맞추고, viseme 스무딩(attack/release, low-pass)과 Emotion 이징으로 표정/입모양의 미세 튐을 억제합니다. Blink 정상화·Micro-Idle·UI Reveal은 대기→청취→발화 전환 구간을 메워 상호작용의 연속성을 확보합니다.

시작 상태는 WBP_Subtitle → WBP_Chat 순 레이어링과 커서/클릭/오버 활성화로 즉시 조작 가능하게 열어 두며, 텍스트 파이프라인에서는 어댑터 단계 정제로 불필요한 문자열이 음성으로 읽히지 않도록 합니다.

Roadmap

11월 — 시스템 품질 정비

  • STT 입력 도입(청취 상태·침묵 타임아웃·발화 전환 규칙)
  • Gaze Sync 1차(카메라 랜드마크 u,v → 아이/헤드 본 매핑 + 기본 스무딩)
  • UI 상태 체계화: Listening / Thinking / Speaking, 백채널 패턴 라이브러리화
  • 전체 성능 개선
  • LLM 교체 가능 구조(어댑터/프롬프트 프로필 분리)
  • 반응 속도 개선(청크 결합, 자막/발화 페이싱 버짓 도입)
  • Lip-sync 튐 완화(viseme attack/release·LPF·이징)
  • 표정 개선(Emotion 커브 시간상수·전이 규칙 정립)

12월 — 패키징 & 평가

  • 패키징/런처·프리셋 번들(데모/전시용 프리셋, 로깅 옵션)
  • 사용자 소규모 평가(파일럿): 존재감·불쾌부조화·이해도 지표 설계 및 1차 수집
  • 공모전 제출: Creative Award 등 출품 자료 정리

1월 — PoC & 현장 운용 설계

  • PoC 기획/세팅 가이드(하드웨어, 카메라/마이크, 네트워크, 공간 조도)
  • 운영 시나리오(오프라인 폴백, 정책/시나리오 롤백, 로그/리플레이)
